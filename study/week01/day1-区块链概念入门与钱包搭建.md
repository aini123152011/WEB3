#### 📝 关键概念笔记

**1. 什么是区块链？**
```
区块链 = 分布式账本
- 每个区块包含多笔交易
- 区块通过哈希值链接
- 数据不可篡改
- 去中心化存储

区块结构示例：
┌─────────────────────┐
│ Block #1000         │
│ ├─ Previous Hash    │──┐
│ ├─ Timestamp        │  │
│ ├─ Transactions     │  │
│ │  ├─ Tx 1         │  │
│ │  ├─ Tx 2         │  │
│ │  └─ Tx 3         │  │
│ └─ Nonce            │  │
└─────────────────────┘  │
                         ↓
┌─────────────────────┐  │
│ Block #1001         │  │
│ ├─ Previous Hash ───┼──┘
│ ├─ Timestamp        │
│ ├─ Transactions     │
│ └─ Nonce            │
└─────────────────────┘
```

**2. 哈希函数 (Hash Function)**
```
哈希特性：
✓ 固定长度输出
✓ 单向不可逆
✓ 雪崩效应（微小改变导致完全不同的输出）

示例：
输入: "Hello World"
SHA-256输出: a591a6d40bf420404a011733cfb7b190d62c65bf0bcda32b57b277d9ad9f146e

输入: "Hello World!" (仅加了感叹号)
SHA-256输出: 7f83b1657ff1fc53b92dc18148a1d65dfc2d4b1fa3d677284addd200126d9069
                     ↑ 完全不同！
```

**3. 工作量证明 (Proof of Work)**
```
挖矿过程：
1. 收集待确认交易
2. 尝试不同的Nonce值
3. 计算区块哈希
4. 检查是否满足难度要求（如前导零的数量）
5. 找到符合条件的哈希后广播

难度示例：
要求: 哈希值前4位必须是0
Nonce = 12345 → Hash = 1a3f5b... ❌
Nonce = 12346 → Hash = 9b2c1e... ❌
Nonce = 72849 → Hash = 0000ab... ✅ 找到了！
```

**4. 双花问题 (Double Spending)**
```
问题: 如何防止同一笔钱被花两次？

解决方案: 区块链共识
- 最长链原则
- 51%攻击成本极高
- 确认时间越长越安全（通常6个确认）
```

#### 📝 以太坊核心概念

**1. 以太坊 vs 比特币**

| 对比维度 | 比特币 | 以太坊 |
|---------|--------|--------|
| **定位** | 数字货币 | 去中心化计算平台 |
| **脚本语言** | 简单脚本（非图灵完备） | Solidity（图灵完备） |
| **区块时间** | ~10分钟 | ~12秒 |
| **共识机制** | PoW（工作量证明） | PoS（权益证明，2022年后） |
| **主要用途** | 价值转移 | 智能合约、DApp、DeFi |
| **账户模型** | UTXO模型 | 账户/余额模型 |

**2. 以太坊账户类型**

```
以太坊有两种账户：

1. 外部账户 (EOA - Externally Owned Account)
   - 由私钥控制
   - 可以发起交易
   - 你的MetaMask钱包就是EOA
   
   结构：
   ┌──────────────────────┐
   │ Address: 0x123...    │
   │ Balance: 1.5 ETH     │
   │ Nonce: 42            │
   └──────────────────────┘

2. 合约账户 (Contract Account)
   - 由代码控制
   - 被动响应交易
   - 包含智能合约代码
   
   结构：
   ┌──────────────────────┐
   │ Address: 0xabc...    │
   │ Balance: 0 ETH       │
   │ Code: {...}          │
   │ Storage: {...}       │
   └──────────────────────┘
```

**3. Gas机制**

```
为什么需要Gas？
- 防止无限循环攻击
- 激励矿工/验证者
- 资源分配机制

Gas计算：
总费用 = Gas Used × Gas Price
示例: 21000 Gas × 50 Gwei = 0.00105 ETH

单位换算：
1 ETH = 1,000,000,000 Gwei (10^9)
1 ETH = 1,000,000,000,000,000,000 Wei (10^18)
```

**4. 交易结构**

```javascript
// 以太坊交易包含的字段
{
  from: "0x1234...",      // 发送方地址
  to: "0x5678...",        // 接收方地址（合约调用则为合约地址）
  value: "1000000000",    // 转账金额（Wei）
  gas: 21000,             // Gas限制
  gasPrice: "50000000000",// Gas价格（Wei）
  nonce: 5,               // 交易序号（防止重放）
  data: "0x...",          // 调用数据（转账为空，合约调用包含函数数据）
  chainId: 1,             // 链ID（1=主网，11155111=Sepolia测试网）
}
```

#### ✍️ 自测题 2
1. 以太坊的Gas机制解决了什么问题？
2. EOA和合约账户的主要区别是什么？
3. 为什么转账简单交易需要21000 Gas，而复杂的合约调用可能需要几十万Gas？

<details>
<summary>点击查看参考答案</summary>

1. 防止死循环/DoS攻击，激励验证者，实现资源定价
2. EOA由私钥控制可主动发交易，合约账户由代码控制只能被动响应
3. 不同操作消耗不同Gas：存储操作最贵，计算次贵，读取最便宜。复杂合约需要更多计算和存储
</details>

### 2.5 使用Etherscan区块浏览器 (30分钟)

#### Etherscan界面详解

**访问你的交易：**
```
URL格式：
https://sepolia.etherscan.io/tx/0x[交易哈希]

示例：
https://sepolia.etherscan.io/tx/0xabc123...
```

**交易详情页面解读：**

```
┌──────────────────────────────────────────┐
│ Transaction Details                       │
├──────────────────────────────────────────┤
│ Transaction Hash:                         │
│ 0xabc123def456...                        │  ← 交易唯一标识
│                                           │
│ Status: ✓ Success                        │  ← 交易状态
│                                           │
│ Block: 4829361                           │  ← 所在区块
│ Block Confirmations: 12                  │  ← 确认数（越多越安全）
│                                           │
│ Timestamp: 2024-01-15 10:23:45          │  ← 时间戳
│                                           │
│ From:                                     │
│ 0x742d35Cc6634C0532925a3b844E291e6A7E834│  ← 发送方
│                                           │
│ To:                                       │
│ 0x8e3c4f2b9a1d0e7c6...                  │  ← 接收方
│                                           │
│ Value: 0.1 ETH                           │  ← 转账金额
│                                           │
│ Transaction Fee: 0.000042 ETH            │  ← Gas费用
│ Gas Price: 2 Gwei (0.000000002 ETH)    │  ← Gas单价
│                                           │
│ Gas Limit & Usage by Txn:               │
│ 21,000 | 21,000 (100%)                  │  ← Gas使用情况
│                                           │
│ Nonce: 0                                 │  ← 交易序号
└──────────────────────────────────────────┘
```

**关键字段说明：**

```
1. Transaction Hash (交易哈希)
   - 交易的唯一标识符
   - 64位十六进制字符串
   - 不可伪造

2. Status (状态)
   ✓ Success: 交易成功
   ✗ Failed: 交易失败（但Gas费已扣除）
   ⏳ Pending: 等待确认

3. Block Confirmations (区块确认数)
   - 表示有多少个新区块在这个交易之后
   - 确认数越多越安全
   - 一般12个确认后认为不可逆

4. Gas相关
   - Gas Limit: 你愿意支付的最大Gas
   - Gas Used: 实际使用的Gas
   - Gas Price: 每单位Gas的价格
   - Transaction Fee = Gas Used × Gas Price

5. Nonce
   - 从该地址发出的交易序号
   - 从0开始递增
   - 防止交易重放攻击
```

## 🆘 常见问题FAQ

### Q1: MetaMask安装后找不到图标？
```
解决方法：
1. 点击Chrome地址栏右侧的拼图图标
2. 找到MetaMask，点击📌图标固定到工具栏
3. 刷新页面
```

### Q2: 领取测试币一直失败？
```
原因可能：
- 该水龙头暂时无币
- 网络拥堵
- 已达到每日限额

解决方法：
1. 尝试其他水龙头网站
2. 等待几小时后重试
3. 检查是否已经到账（刷新钱包）
4. 加入Web3学习社区求助
```

### Q3: 交易一直Pending怎么办？
```
原因：
- Gas价格设置过低
- 网络拥堵

解决方法：
1. 等待（测试网通常5-10分钟）
2. 加速交易：
   - 在MetaMask中点击该交易
   - 选择 "Speed Up"
   - 提高Gas价格
3. 取消交易：
   - 点击 "Cancel"
   - 支付少量Gas取消
```

### Q4: 不小心关闭了MetaMask，助记词找不到了？
```
找回方法：
1. 打开MetaMask
2. 点击右上角设置⚙
3. 安全与隐私 → 显示助记词
4. 输入密码
5. 重新抄写备份

⚠️ 警告：只在安全的私人环境中操作！
```

### Q5: 余额显示不对？
```
解决方法：
1. 确认已切换到Sepolia网络
2. 刷新MetaMask：
   设置 → 高级 → 重置账户
3. 在Etherscan手动查询地址余额
4. 清除浏览器缓存后重启
```
